## Coding Toy Buyer's Guide

### Screenless vs. Screen-Based Coding Toys

| Type | Best For | Examples |
|------|---------|---------|
| Screenless | Ages 4–8, screen-time-concerned families | Coding Critters, Botley 2.0 |
| Hybrid (physical + screen) | Ages 5–10 | Osmo Coding Kit |
| App-based | Ages 6–14 | Dash Robot, LEGO Spike |

### The Coding Learning Ladder

The best approach is a progression: start with screenless toys (Coding Critters) at ages 4–6, move to hybrid physical-app (Osmo) at ages 6–8, progress to full app coding (Dash, LEGO Spike) at ages 8+, then transition to text-based code (Scratch, Python) at 10+. Each step builds on the last without overwhelming the learner.

### What to Look For

**Good coding toys** teach concepts that transfer: sequencing (put commands in the right order), loops (repeat actions efficiently), conditionals (if X, do Y), and debugging (find what's wrong and fix it). Avoid toys that only offer a pre-built experience with no genuine programming — the child should be creating behavior, not just watching it.
